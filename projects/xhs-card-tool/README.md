# 小红书图文卡片编辑器

一个手机优先的小红书中文图文卡片排版工具。它解决的是很具体的问题：把一段中文长文快速整理成固定尺寸的图文卡片，方便预览、微调和导出发帖素材。

当前版本是 MVP，适合本地运行和个人工作流使用。项目会优先保持简单、稳定、可改，而不是做成复杂的设计软件。

## 功能

- 粘贴中文长文，一键处理正文。
- 自动清理多余空行和不必要的分行。
- 自动识别首行标题和编号小标题。
- 自动生成加粗和高亮重点，并限制每页最多 1 个加粗、1 个高亮。
- 支持手动编辑处理后的 Markdown。
- 支持修改作者头像、作者名和生成日期。
- 支持插入本地图片或远程图片 URL。
- 自动分页为固定比例卡片。
- 导出 `1080 x 1440` PNG，适合小红书图文帖。

## 使用流程

1. 打开工具页面。
2. 在正文输入框粘贴原始文字。
3. 点击 `处理正文`。
4. 在下方预览卡片效果。
5. 如有需要，展开 Markdown 编辑区微调标题、加粗、高亮、图片和分页。
6. 上传头像或正文图片，修改作者名。
7. 点击 `导出`，下载 PNG 或 ZIP。

## 本地运行

需要 Node.js 和 npm。

```bash
npm install
npm run dev
```

如果要在手机上编辑，电脑和手机需要在同一个网络下。启动后使用 Vite 输出的 Network URL 在手机浏览器打开。

常见启动问题：

```bash
pwd
cd projects/xhs-card-tool
npm run dev
```

如果 npm 报 `uv_cwd` 或 `process.cwd failed`，通常是终端当前目录被移动或没有权限。重新 `cd` 到项目目录后再运行即可。

## Markdown 语法

处理后的正文可以继续手动编辑：

```markdown
# 主标题

## 小标题

这是普通正文，**这里会加粗**，==这里会高亮==。

![图片说明](图片ID或图片URL)

---

这里会从新的一页开始。
```

说明：

- `#` 会渲染为主标题。
- `##` 会渲染为小标题。
- `**文字**` 会渲染为加粗。
- `==文字==` 会渲染为高亮。
- `---` 会强制分页。
- `![说明](图片ID或URL)` 会插入正文图片。

## 自动处理规则

工具不会重写你的观点，也不会替你补充新内容。自动处理只做排版层面的整理：

- 合并被错误断开的中文行。
- 保留英文单词之间的空格。
- 识别编号小标题，例如 `1、标题`。
- 对明显的短重点做轻量加粗或高亮。
- 每页最多自动生成 1 个加粗和 1 个高亮，避免整页过度标记。

如果你想完全控制样式，可以在处理后 Markdown 里手动删除或调整 `**` 和 `==`。

## 导出规格

- 单张尺寸：`1080 x 1440`
- 比例：`3:4`
- 多页时导出 ZIP。
- 页面不会包含假的小红书状态栏、点赞、评论、关注等 UI。

## 项目结构

```text
src/
  App.tsx                    # 主界面和交互
  components/CardPage.tsx    # 卡片预览和导出视图
  domain/
    textProcessor.ts         # 文本清洗和自动强调
    markdownParser.ts        # Markdown 到卡片块
    paginator.ts             # 固定尺寸分页估算
  services/
    exportService.ts         # PNG / ZIP 导出
    projectStorage.ts        # 本地草稿保存
```

## 隐私

当前版本在本地浏览器运行，正文、头像和图片默认保存在本地浏览器存储里，不会主动上传到服务器。

如果你使用远程图片 URL，图片加载会请求对应的远程地址。
